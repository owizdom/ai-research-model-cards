# Article data — chart-ready CSVs

Seven CSVs corresponding to the seven proposed charts for the
"AI Model Safety Cards" post. Each CSV is one chart's worth of data.

Generated by: `charts/build_article_data.py` (re-run any time to regenerate).

## What's real vs. what's estimated

| File | Purpose | Provenance | Safe to publish as-is? |
|---|---|---|---|
| **D_benchmark_uniqueness.csv** | The 89%-by-1-lab L-curve | **Real** — computed from snapshot_2026-04-28.json | ✅ Yes |
| **E_card_word_counts.csv** | The 100× spread | **Real** — pulled from live API word-count timeline | ✅ Yes |
| **G_extraction_recall.csv** | Methodology validator | **Mixed** — old-pipeline numbers from system_audit.md (real); new-pipeline numbers are estimates by lab (`mean_recall_pct` etc. should be re-measured before publication) | ⚠️ Anchors are real; per-lab/year cells need validation |
| **C_card_timeline.csv** | The 50-card timeline | **Mixed** — `word_count` and `n_benchmarks` are real; `register` and `first_mention_*` are heuristic | ⚠️ Heuristic columns need manual validation |
| **A_threshold_vs_uplift.csv** | Approaching-threshold alarm | **Best-effort** — populated from public-knowledge reading of each card. `confidence` column flags which rows quote verbatim vs. paraphrase | ❌ Validate every row against the actual card text before publication |
| **B_language_register.csv** | Vocabulary convergence | **Best-effort** — heuristic classifier by (lab, year). Three buckets: reassurance / threshold / hedging | ❌ Should be replaced with manual classification or LLM-classified-with-rubric |
| **F_limitations_vocabulary.csv** | Candor expansion over time | **Best-effort** — counts of cards mentioning each limitation term per year, estimated. Easy to replace with grep-based counts on raw card text | ❌ Replace with grep-based counts before publication |

## Files

### A_threshold_vs_uplift.csv (14 rows)
One row per published threshold-vs-measurement pair. Columns:
- `lab`, `document_slug`, `release_date`, `risk_category`, `framework`
- `threshold_value` / `threshold_unit` — the lab's declared acceptable bound
- `measured_value` / `measured_unit` — the actual measurement disclosed
- `source_quote`, `confidence`

The chart: time-series with two lines per lab, threshold and measured uplift, converging by 2026.

### B_language_register.csv (50 rows, one per card)
One row per card classified into reassurance / threshold / hedging. Columns:
- `document_slug`, `lab`, `release_date`, `year`
- `register` — one of three buckets
- `evidence_phrase` — short phrase justifying classification
- `confidence` — flag for `low (heuristic)` vs `med (validated)`

The chart: stacked area / stream chart over time, three colors. Reassurance dominant 2023, threshold dominant 2024-25, hedging emerges 2025-26.

### C_card_timeline.csv (50 rows)
Comprehensive per-card row. Columns:
- `document_slug`, `lab`, `release_date`, `release_confidence`
- `word_count` (real), `n_benchmarks` (real)
- `register` (heuristic)
- `first_mention_catastrophic`, `first_mention_cot_unfaithfulness`, `first_mention_cannot_rule_out` (outline-seeded)
- `data_provenance` — column noting per-row what's real vs. heuristic

The chart: horizontal strip plot, x = release date, y = lab, bar height = word count, glyphs for first-mentions.

### D_benchmark_uniqueness.csv (5 rows)
One row per "n labs reporting." Columns:
- `n_labs_reporting`, `n_benchmarks`, `pct_of_total`, `example_benchmarks`

The chart: L-curve histogram. Massive bar at "1 lab" (276 benchmarks, 88.7%), tapering to "5 labs" (1 benchmark — MMLU). No benchmark reported by all 6.

**Sanity check**: matches outline claim of "89% reported by only one lab."

### E_card_word_counts.csv (50 rows, sorted descending by word_count)
One row per card. Columns:
- `document_slug`, `lab`, `release_date`, `release_confidence`, `word_count`

The chart: horizontal lollipop or strip plot, sorted by word count, colored by lab. Anthropic Mythos at top (65,849 words); Mistral 7B at bottom (745 words). Span ≈ 88×.

**Sanity check**: matches outline claim of "750 to 68,000 words" (slight off because of word-count methodology — close enough).

### F_limitations_vocabulary.csv (24 rows: 6 terms × 4 years)
One row per (limitation_term, year). Columns:
- `limitation_term` — one of: hallucination, bias, cot_unfaithfulness, reward_hacking, deception, benchmark_saturation
- `year` — 2023 to 2026
- `n_cards_mentioning`, `n_total_cards`, `pct_mentioning`
- `example_card_slugs`, `confidence`

The chart: small-multiples line chart, one panel per term, `pct_mentioning` over time. Hallucination and bias plateau / decline; CoT unfaithfulness, reward hacking, deception, benchmark saturation emerge 2024-2026.

### G_extraction_recall.csv (12 rows: 6 labs × 2 pipelines)
One row per (lab, pipeline). Columns:
- `lab`, `year`, `pipeline` (old 14k window vs new 30k window)
- `n_cards`, `mean_recall_pct`, `recall_low_pct`, `recall_high_pct`
- `n_evals_extracted`, `notes`

The chart: bar chart with error bars, lab on x, recall on y, two bar groups per lab (old vs new pipeline). Shows the recall floor lifted by the pipeline upgrade — inoculates the post against "have you accounted for extraction recall?"

## How to regenerate

```bash
python3 charts/build_article_data.py
```

Reads `charts/data/snapshot_2026-04-28.json`, `charts/release_dates.csv`,
and the live API for word counts. Outputs to this folder.

## Pre-publication validation checklist

Before publishing the article, manually validate the best-effort CSVs:

- [ ] **A** — Open each card cited; confirm threshold and measurement values match. Replace any `confidence: med (paraphrased)` with verbatim quotes or remove the row.
- [ ] **B** — Sample 10 cards, manually classify register, compare to heuristic. Aim for ≥ 80% agreement before trusting.
- [ ] **F** — Replace estimated counts with grep on raw card text. The `data/raw_card_text/` directory (if it exists) is the source.
- [ ] **G** — Re-run extraction recall measurement on 5 cards from each (lab, pipeline) combo to validate the per-cell estimates.
